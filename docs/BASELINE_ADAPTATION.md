# Baseline Adaptation Protocol

This document defines how each baseline family should be adapted to the project-level native-action CCRCov evaluation protocol. The purpose is to preserve each method's original action semantics while still comparing all methods with the same dataset split, frozen teacher, K, cost threshold, and cost function.

| Baseline | Original Output | Native Action | Main Evaluation | Notes |
| -------- | --------------- | ------------- | --------------- | ----- |
| Ours | class-level selected subgraphs | hard deletion $G \setminus s$ | native-action CCRCov | deletion-only by design |
| GCFExplainer | full counterfactual graph set | choose counterfactual graph $C$ | native full-graph CCRCov | do not force subgraph projection |
| GlobalGCE | graph transformation rule LHS->RHS | apply rule replacement/addition/deletion | native rule CCRCov | SuppCov only applies to LHS if needed |
| RLHEX | full CF molecule or global CF rule | depends on output type | native full-graph or rule CCRCov | if local outputs, aggregate pool then greedy top-K |
| CF-GNNExplainer | local edge deletion | edge deletion | pool local deletions then greedy top-K | local flip does not imply global flip |
| CLEAR | local counterfactual graph | full CF graph or diff action | pool local CFs then greedy top-K | projected deletion only auxiliary |
| SME / BRICS-Mask | important fragment / mask score | hard deletion of selected fragment | deletion CCRCov | original method may be factual, so must re-evaluate CF effect |

## 2026-06 CCRCov Evaluation Protocol

The following rules are mandatory for baseline adaptation:

1. Do not assume local CF flip rate is 100% under our evaluation. A local counterfactual generated by a baseline may fail under the shared dataset-level teacher, shared split, or shared cost threshold.
2. Local flip rate is diagnostic only. It can be reported for method debugging, but it is not the final cross-baseline result.
3. The main metric is global top-K CCRCov.
4. Full-graph baselines should not be forced into SuppCov or subgraph projection.
5. Addition / replacement baselines are allowed to use their native edits.
6. Fairness is ensured by the shared frozen teacher, shared dataset split, shared K, shared threshold, and shared cost function.
7. For methods producing only one local CF per input graph, construct a candidate pool, canonicalize outputs, deduplicate candidates, and select top-K by greedy marginal CCRCov gain.

## Native-action Principle

Each baseline should be evaluated through the action it was designed to perform:

- deletion methods delete atoms, bonds, fragments, or edges;
- full-graph CF methods propose or choose a complete counterfactual graph;
- rule methods apply graph transformation rules;
- addition / replacement methods execute their additions or replacements before evaluation;
- mask methods are converted to the documented intervention implied by the mask, usually hard deletion for deletion-compatible auxiliary analysis.

This protocol avoids rewarding or penalizing a baseline simply because its original output type is not a subgraph. The common comparison point is the intervened graph $G_i^a = T_a(G_i)$ and its low-cost counterfactual effect under the frozen teacher.

## Local-to-global Pooling

Some baselines naturally produce one local counterfactual per input graph rather than a class-level global rule set. These methods should be adapted through:

```text
local CF outputs -> candidate pool -> canonicalization -> deduplication -> greedy top-K
```

The greedy step chooses candidates that maximize marginal CCRCov gain on the evaluation subset used for candidate selection. Final reporting must still use the agreed split protocol and should clearly state whether the greedy selection was performed on validation or test data.

## Auxiliary Metrics

Original paper metrics for each baseline may be recorded for traceability, but they must not be used as final horizontal comparison results. Final horizontal comparison must use the unified teacher re-evaluation.

SuppCov is temporarily reserved for subgraph-specific auxiliary analysis. StructRed, MatchRate, and DeleteValidRate are also auxiliary and only apply to compatible output types.
